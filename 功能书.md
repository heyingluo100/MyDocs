# 个人文档库 — 功能书

## 功能清单

### 一、内容管理系统（文件夹即标签）

| # | 功能点 | 说明 | 状态 |
|---|--------|------|------|
| 1.1 | 文件夹即标签 | `content/` 下每个文件夹名就是分类标签 | ✅ 已实现 |
| 1.2 | Word .docx 支持 | `.docx` 文件通过 mammoth.js 构建时转为 HTML，精确保留空行 | ✅ 已实现 |
| 1.3 | Word .doc 支持 | `.doc`（旧版 Word 97-2003）通过 word-extractor 提取文本渲染 | ✅ 已实现 |
| 1.4 | Markdown 支持 | `.md` 文件通过 markdown-it 构建时转为 HTML | ✅ 已实现 |
| 1.5 | 纯文本支持 | `.txt` 文件内容直接渲染在页面上（非附件下载） | ✅ 已实现 |
| 1.6 | 附件文件支持 | PDF/Excel 等文件自动复制到 public/files/ 供预览 | ✅ 已实现 |
| 1.7 | 构建脚本 | `scripts/build-content.js` 自动扫描分类文件夹 → 生成 JSON 索引 | ✅ 已实现 |
| 1.8 | 内容 Base64 编码 | 构建时 HTML 内容 Base64 编码，运行时解码 | ✅ 已实现 |
| 1.9 | Vite 构建钩子 | `npm run dev/build` 时自动执行构建脚本 | ✅ 已实现 |
| 1.10 | 空格/换行保留 | CSS `white-space: pre-wrap` + docx `transformDocument` 精确保留段落间距 | ✅ 已实现 |
| 1.11 | 跳过临时文件 | 自动跳过 Word 临时文件（`~$` 开头），避免构建报错 | ✅ 已实现 |
| 1.12 | 合集（子文件夹） | 标签文件夹下的子目录自动识别为合集，合集内文章按文件名排序 | ✅ 已实现 |
| 1.13 | 多标签（.tags 文件） | 合集内放 `.tags` 文件（不带前缀）或文件旁放 `文件名.tags`（带前缀）指定额外标签 | ✅ 已实现 |
| 1.14 | 尾部空行裁剪 | 构建时自动裁掉文档末尾空段落，保留文中空行 | ✅ 已实现 |
| 1.15 | 双时间字段 | 创建时间（birthtime/.docx 元数据）+ 更新时间（mtime），按创建时间排序 | ✅ 已实现 |

## 文件结构总览

```
E:/个人网站/
├── content/                    # ← 文件夹即标签，拖入文件即发布
│   ├── 工作/                   #    标签「工作」
│   │   └── 报告.docx           #    Word 文件自动转为网页
│   ├── 技术笔记/               #    标签「技术笔记」
│   │   └── Vue3最佳实践.md     #    Markdown 也支持
│   └── 公告/
│       └── 欢迎.md
├── public/                     # 静态资源
├── src/
│   ├── components/             # Vue 组件（6 个）
│   ├── composables/            # Composables（4 个）
│   ├── views/                  # 视图页面（3 个）
│   ├── data/articles.json      # 构建生成的文章索引
│   ├── styles/main.css         # 全局样式 + 保护 CSS
│   ├── router/index.js         # 路由配置
│   ├── App.vue                 # 根组件
│   └── main.js                 # 入口文件
├── scripts/build-content.js    # 内容构建脚本
├── PRD.md                      # 产品需求文档
├── 功能书.md                    # 本文件
├── index.html
├── vite.config.js
├── package.json
└── start.bat
```
