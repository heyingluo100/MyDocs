| 1.17 | 单篇锁 | 文章旁放 `文件名.lock` 文件标记锁定，每篇可设独立邀请码，内容 AES-256-GCM 加密 | ✅ 已实现 |

| 1.22 | 锁状态可视化 | 首页卡片/合集卡片/导航/目录统一锁图标：未解锁 amber 闭合锁，已解锁 green 打开锁 | ✅ 已实现 |
| 1.23 | 导航锁检查 | 上下篇/目录跳转锁定文章时弹出确认提醒，已解锁则直接跳转 | ✅ 已实现 |
| 1.24 | 锁定文章导航可见 | 文章锁定时底部导航和目录按钮仍然可见可操作 | ✅ 已实现 |
| 1.25 | 锁定内容加密 | 锁定文章内容使用 **AES-256-GCM 加密**存储，密钥由邀请码经 **PBKDF2-SHA256/迭代 200000 次 + 每篇独立 salt** 派生（v2 协议，兼容 v1 旧产物），lockHash 不暴露在客户端 JSON 中 | ✅ 已实现 |
| 1.26 | 暴力破解防护 | 邀请码连续输错每次指数退避（1s→2s→4s→8s），第 5 次错锁定 5 分钟。**按 slug 独立计数**，A 文章被锁不影响 B 文章；全站锁单独一条计数器 | ✅ 已实现 |
