# 操作指南 · 快速上手

> 这一篇教你从零把网站跑起来：拉代码 → 本地跑 → 推 GitHub → 部署上线。
>
> **如果你完全没用过 GitHub、没装过 Node.js / Git** —— 先看[环境准备文档](01-环境准备.md)把账号注册、软件安装搞定，再回来。
>
> **如果你已经有 GitHub 账号、本地装了 Node 和 Git** —— 直接从下面开始。

---

## 一、整体链路

整个项目的工作流是这条链：

```
本地电脑                    GitHub                   Netlify / Vercel / GitHub Pages
┌─────────┐    git push    ┌────────┐    自动构建    ┌──────────────────┐
│ 写文档  │ ─────────────→ │ 仓库   │ ────────────→ │ 你的网站（公开） │
│ npm run │                │ 代码 + │                │  xxx.netlify.app  │
│ build   │                │ 文档   │                │  xxx.vercel.app   │
└─────────┘                └────────┘                └──────────────────┘
```

- **你只在本地动文档**（往 `content/` 里加 / 改文件）
- **每次想发布就推一下** GitHub
- **网站会自动重新部署**，1-2 分钟后访问者就能看到新内容

整条链一次配好后，日后只剩一条命令：`npm run build && git add . && git commit -m "更新" && git push`。

---

## 二、核心链路（实操）

> 第一次走这章会比较慢（要拉代码、配关联、首次推送、配部署）。配好之后日常更新只看 [日常使用文档](03-日常使用.md)。

### 2.1 拉代码到本地

打开终端，逐行输入：
```
git clone https://github.com/heyingluo100/MyDocs.git my-docs && cd my-docs
```

> 这会把项目模板下载到你电脑上的 `my-docs` 文件夹里。
>
> **没装 Git？** 见[环境准备文档 5.5 节](01-环境准备.md)装一下再回来。

### 2.2 安装依赖 + 本地预览

```
npm install
```

等待安装完成（可能需要几分钟，看到没有红色报错就行）。

> **遇到红色报错怎么办？** 别慌，把报错信息**完整复制**（从第一行 `npm error` 或 `npm ERR!` 一直到最后），丢给 AI 工具就行：

>- **Monica**：https://monica.im
> - **DeepSeek**：https://chat.deepseek.com
> - **通义千问**：https://tongyi.aliyun.com
>
> 提问模板：
> ```
> 我在 Windows / Mac 上跑 npm install 报错，请帮我看看怎么解决，下面是完整报错：
>
> [把报错全部粘贴到这里]
> ```
>
> AI 会告诉你原因（一般是网络慢、Node 版本不对、权限不够等）和具体的修复命令。照它说的做，再 `npm install` 重试一遍。

然后本地启动：
```
npm run dev
```

浏览器打开终端显示的地址（一般是 `http://localhost:5173`）就能看到网站。文件有变动会自动更新，不用手动刷新。

> **没装 Node.js？** 见[环境准备文档 5.4 节](01-环境准备.md)装一下再回来。
>
> Windows 用户可以直接双击项目根目录的 `start.bat`，效果一样。

### 2.3 创建 content 目录 + 写第一篇文档

`content/` 是你放所有文档的地方，**不会上传到 GitHub**（在 `.gitignore` 里），属于你自己的私有内容。

**方式一：图形界面（推荐新手）**

打开项目文件夹（刚才 clone 下来的 `my-docs`），在里面**右键 → 新建文件夹**，把文件夹命名为 `content`。

**方式二：命令行**
```
mkdir content
```

两种方式效果完全一样，挑顺手的就行。

然后在 `content/` 里创建一个**子文件夹**（就是分类标签，同样**右键新建**就行），把你的文档放进去。例如：

```
content/
└── 公告/
    └── 欢迎.md     ← 内容随便写点
```

刷新浏览器就能看到这篇文档出现在网站上。

### 2.4 关联到你自己的 GitHub 仓库

刚才 clone 下来的代码默认连到模板仓库，你要把它改成连到你自己的仓库。

**先建仓库**：
1. 登录 GitHub，点右上角 **+** → **New repository**
2. 填写：
   - Repository name：随便取个名字（如 `my-docs`）
   - 选 **Private**（强烈建议私有，避免别人 fork 拉你的 `.lock` 哈希做撞库）
   - **不要**勾选任何初始化选项（不要勾 README、.gitignore）
3. 点 **Create repository**
4. 复制页面上显示的仓库地址（如 `https://github.com/你的用户名/my-docs.git`）

**改本地关联**：
```
git remote set-url origin https://github.com/你的用户名/my-docs.git
```

> 把上面的地址换成你刚复制的那个。
>
> **没注册 GitHub？** 见[环境准备文档 5.1 节](01-环境准备.md)再回来。

### 2.5 首次构建并推送

```
npm run build
git add .
git commit -m "首次提交"
git push -u origin master
```

> **建议一行一行复制粘贴执行**，不要四条一起糊上去。这样如果某一步报错，你能直接知道是哪一步出的问题——把那一行的命令 + 完整报错信息一起丢给 AI（DeepSeek / 千问 / Monica，URL 见 2.2 节），它能精准告诉你怎么改。
>
> 如果四条一起跑出错了，连哪步崩的都不知道，AI 也只能猜。

> 如果提示输入用户名密码，输入你的 GitHub 用户名和 **Personal Access Token**（不是登录密码）。
>
> 获取 Token：GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token，勾选 `repo` 权限。

推送成功后，去 GitHub 仓库页面应该能看到代码已经上传。

### 2.6 部署到 Netlify（推荐）

**为什么推荐 Netlify**：国内可直接登录管理后台，不用翻墙；免费套餐每月 15 GB 流量对个人网站完全够用。

1. 打开 https://app.netlify.com
2. 点 **Add new site** → **Import an existing project**
3. 选 **Deploy with GitHub**，授权后找到你刚推送的仓库（`my-docs`）
4. 构建配置（如果没自动填好就手动填）：
   - **Build command**：`npm run build`
   - **Publish directory**：`dist`
5. 点 **Deploy site**，等 1-2 分钟，部署完成
6. Netlify 会给你一个网址（如 `https://random-name-xxx.netlify.app`），这就是你的网站
7. 想改这个二级域名：进站点 → **Site configuration** → **Change site name**，改成你喜欢的（如 `my-docs.netlify.app`）

以后每次 `git push`，Netlify 自动重新部署。

> **关于 SPA 路由**：项目里已经放了 `public/_redirects`（一行 `/*  /index.html  200`），Netlify 自动识别，刷新内页不会出现 404。这个文件不用动。
>
> **没注册 Netlify？** 见[环境准备文档 5.2 节](01-环境准备.md)再回来。

### 2.7 部署到 Vercel（备选）

1. 打开 https://vercel.com/dashboard
2. 点 **Add New...** → **Project**
3. 找到你刚推送的仓库（`my-docs`），点 **Import**
4. 设置页面保持默认，直接点 **Deploy**
5. 等 1-2 分钟，部署完成
6. Vercel 给你一个网址（如 `https://my-docs-xxx.vercel.app`），这就是网站地址（前缀可改，后缀 `.vercel.app` 不能改）

以后每次 `git push`，Vercel 自动重新部署。

> **没注册 Vercel？** 见[环境准备文档 5.3 节](01-环境准备.md)再回来。

### 2.8 部署到 GitHub Pages（备选）

1. 打开你的 GitHub 仓库页面
2. 点 **Settings** → 左侧 **Pages**
3. Source 选择 **GitHub Actions**
4. 项目已经包含了自动部署的配置文件（`.github/workflows/deploy.yml`），推送代码后会自动构建部署
5. 等几分钟，网站地址为 `https://你的用户名.github.io/my-docs/`

### 2.9 自定义域名

三种部署平台都支持绑定自己的域名（如 `docs.example.com`），步骤大同小异。

#### Netlify
1. 进入站点 → **Domain management** → **Add a domain**
2. 输入域名 → **Verify** → **Add domain**
3. 去域名注册商后台加 DNS 记录：
   - 子域名（如 `docs.example.com`）：**CNAME** 记录，值为 Netlify 显示的目标（通常是 `<你的站点>.netlify.app`）
   - 根域名（如 `example.com`）：**A** 记录，值为 Netlify 显示的 IP（**以 Netlify 后台实时显示的值为准**）
4. 等 DNS 生效（几分钟到几小时），回 Netlify 点 **Verify DNS configuration**，自动配 HTTPS

#### Vercel
1. 进入项目 → **Settings** → 左侧 **Domains**
2. 输入域名 → **Add**
3. 去域名注册商后台加 DNS 记录：
   - 子域名：**CNAME** 记录，值为 `cname.vercel-dns.com`
   - 根域名：**A** 记录，值为 `76.76.21.21`
4. 等 DNS 生效，Vercel 自动配 HTTPS

#### GitHub Pages
1. 仓库 **Settings** → **Pages** → **Custom domain** 输入域名 → **Save**
2. 域名注册商加 CNAME 记录：`你的用户名.github.io`
3. 等生效，GitHub 自动配 HTTPS

> **没有域名？** 不用买，直接用平台给的免费二级域名就行。

### 2.10 修改网站名称

网站默认叫「文档库」，改名需要改 3 个地方：

| 文件 | 位置 | 改什么 |
|------|------|--------|
| `index.html` | 第 9 行 `<title>` | 浏览器标签页上显示的标题 |
| `src/components/AppHeader.vue` | 搜索「文档库」 | 网站左上角显示的名称 |
| `src/App.vue` | 搜索「Personal Docs」 | 网站底部的版权文字 |

改完后在终端执行下面这条命令，就会自动构建并推送到部署平台：

```
npm run build && git add . && git commit -m "更新网站名称" && git push
```

等 1-2 分钟，Netlify / Vercel / GitHub Pages 自动重新部署，刷新网站就能看到新名称。

> 邀请码验证页面（`src/components/InvitationGate.vue`）上也有一行「Personal Docs」，如果开了全站锁也一起改掉。
