# 操作指南 · 日常使用

> 网站搭好后，日常写文章、加合集、发布更新都看这一篇。
>
> 第一次搭建请先看 [快速上手](02-快速上手.md)。

---

## 一、添加 / 修改文档

1. 在 `content/` 下创建文件夹（文件夹名 = 分类标签）
2. 把 `.docx`、`.md` 或 `.txt` 文件拖进去
3. 完成

**目录结构示例**：
```
content/
├── 工作/                ← 分类「工作」
│   ├── 周报.docx        ← Word 文档
│   └── 项目总结.md      ← Markdown 文档
├── 学习/                ← 分类「学习」
│   └── 笔记.docx
├── 公告/
│   ├── 欢迎.md          ← 单独文章
│   └── 版本记录/        ← 合集（子文件夹 = 合集）
│       ├── v0.1.md
│       └── v0.2.md
└── P3/
    └── P345联动合集/    ← 跨标签合集
        ├── .tags        ← 内容: P4\nP5（额外标签）
        ├── 01-第一章.md
        └── 02-第二章.md
```

---

## 二、合集

在标签文件夹下创建**子文件夹**，即为合集。合集内文章按文件名排序（建议用 `01-`、`02-` 数字前缀控制顺序）。

---

## 三、多标签（标签穿插）

文章或合集需要出现在多个标签下时，创建 `.tags` 文件指定额外标签：

- **合集多标签**：在合集文件夹内新建 `.tags`（文件名就叫 `.tags`），每行写一个额外标签名
- **单独文章多标签**：在同级目录新建 `文件名.tags`（如 `周报.tags` 对应 `周报.docx`），每行写一个额外标签名

**注意**：合集用的是 `.tags`（不带前缀），单独文章用的是 `文件名.tags`（带文件名前缀），别搞混。

---

## 四、本地预览（非必要）

> **这一步不是必须的**——只在你想"本地先看看效果再决定要不要发"时才需要。
>
> 如果你确认改完直接发就行，可以**跳过这步**，直接去看第五章「发布上线」。

```
npm run dev
```

浏览器打开显示的地址即可预览。文件有变动自动更新，无需手动刷新。

适合用本地预览的场景：
- 试改一篇文档的内容、看排版效果
- 调整文件夹结构（合集 / 标签 / 多标签）
- 测试 `.lock` / `.pin` / `.note` 等特殊文件
- 不确定改动效果，怕直接推到线上不好看

---

## 五、发布上线

```
npm run build && git add . && git commit -m "更新文档" && git push
```

推送后 Netlify / Vercel / GitHub Pages 自动部署，1-2 分钟后网站更新。
