# 操作指南

## 一、新电脑首次部署（只需做一次）

### 第 1 步：安装 Node.js
1. 打开 https://nodejs.org
2. 下载 **LTS 版本**（左边那个按钮）
3. 一路下一步安装
4. 打开终端（Windows 按 `Win+R` 输入 `cmd` 回车），输入：
```
node -v
```
看到版本号（如 `v20.x.x`）就成功了。

### 第 2 步：安装 Git
1. 打开 https://git-scm.com
2. 下载安装，一路下一步
3. 终端输入：
```
git -v
```
看到版本号就成功了。

### 第 3 步：下载项目
打开终端，逐行输入：
```
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
```
等依赖安装完成（可能需要几分钟）。

### 第 4 步：创建 content 目录
项目下载后没有 `content/` 目录（它不上传到 GitHub），需要自己建：
```
mkdir content
```
然后在 `content/` 里创建分类文件夹，把文档拖进去即可。

如果你有之前电脑上的 `content/` 文件夹备份，直接复制过来放到项目根目录下。

## 二、日常操作

### 添加 / 修改文档

1. 在 `content/` 下创建文件夹（文件夹名 = 分类标签）
2. 把 `.docx`、`.md` 或 `.txt` 文件拖进去
3. 完成！

**目录结构示例：**
```
content/
├── 工作/                ← 分类「工作」
│   ├── 周报.docx        ← Word 文档
│   └── 项目总结.md      ← Markdown 文档
├── 学习/                ← 分类「学习」
│   └── 笔记.docx
├── 公告/
│   ├── 欢迎.md          ← 单独文章
│   └── 版本记录/        ← 合集（子文件夹 = 合集）
│       ├── v0.1.md
│       └── v0.2.md
└── P3/
    └── P345联动合集/    ← 跨标签合集
        ├── .tags        ← 内容: P4\nP5（额外标签）
        ├── 01-第一章.md
        └── 02-第二章.md
```

### 合集功能

在标签文件夹下创建**子文件夹**，即为合集。合集内文章按文件名排序。

### 多标签（标签穿插）

文章或合集需要出现在多个标签下时，创建 `.tags` 文件指定额外标签：

- **合集多标签**：在合集文件夹内新建 `.tags` 文件（文件名就叫 `.tags`），每行写一个额外标签名
- **单独文章多标签**：在同级目录新建 `文件名.tags` 文件（如 `周报.tags` 对应 `周报.docx`），每行写一个额外标签名

**注意**：合集用的是 `.tags`（不带前缀），单独文章用的是 `文件名.tags`（带文件名前缀），别搞混！

文件只存一份，不需要复制。

**示例**：
```
content/
├── P3/
│   ├── 通知.md              ← 单独文章
│   ├── 通知.tags            ← 内容: P4（通知.md 额外出现在 P4 标签下）
│   └── P345联动合集/
│       ├── .tags            ← 内容: P4\nP5（合集额外出现在 P4、P5 标签下）
│       ├── 01-第一章.md
│       └── 02-第二章.md
```

### 本地预览

```
npm run dev
```

文件有变动会自动更新，无需手动刷新。

### 发布上线（3 步）

```
npm run build && git add . && git commit -m "更新文档" && git push
```
推送后 Vercel 自动部署，1-2 分钟后网站更新。

```
npm run build && git add . && git commit -m "更新文档" && git push
```

## 四、邀请码系统

网站支持两层邀请码保护，默认关闭，按需开启。

### 生成邀请码

```
npm run gen-code
```

输出邀请码（明文，发给受邀用户）和哈希值（填入配置）。也可以自定义码：

```
npm run gen-code "你想要的码"
```

### 开启全站锁

全站锁开启后，访问任何页面都必须先输入邀请码。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 打开 `src/config/invitation.js`，修改：
```js
global: {
  enabled: true,
  codeHash: '粘贴哈希值到这里',
}
```
3. 重新构建发布
4. 把邀请码发给受邀用户

### 锁定单篇文章

单篇锁可以给特定文章设独立邀请码，每篇码可以不同。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 在文章旁创建同名 `.lock` 文件，内容为哈希值：
```
content/
├── 工作/
│   ├── 机密报告.docx
│   └── 机密报告.lock    ← 内容为哈希值（一行）
```
3. 打开 `src/config/invitation.js`，设 `article.enabled: true`
4. 重新构建发布
5. 该文章的专属邀请码发给特定用户

### 锁定整个合集

在合集文件夹内放 `.lock` 文件（和 `.tags` 同级），合集内所有文章自动锁住，共享同一个邀请码。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 在合集文件夹内创建 `.lock` 文件，内容为哈希值：
```
content/
├── 工作/
│   └── 机密合集/
│       ├── .lock            ← 内容为哈希值，整个合集一把锁
│       ├── 01-第一章.md
│       └── 02-第二章.md
```
3. 打开 `src/config/invitation.js`，设 `article.enabled: true`
4. 重新构建发布

如果合集内某篇文章还有自己的 `.lock` 文件（如 `01-第一章.lock`），那篇文章优先用自己的码，其余用合集的码。

### 关闭邀请码

把 `src/config/invitation.js` 里对应的 `enabled` 改为 `false`，删掉 `.lock` 文件（可选），重新构建即可。

### 注意

- 邀请码验证后 **7 天有效**，到期需重新输入
- 全站锁和单篇锁可以同时开启（双层保护）
- 配置文件中只存哈希值，不存明文码
- `.lock` 文件里也是哈希值，可以安全提交到 Git

## 五、文档置顶

支持将重要文档或合集固定在列表顶部，支持全局置顶和标签内置顶。

### 置顶单篇文章

在文章旁创建同名 `.pin` 文件：
```
content/
├── 工作/
│   ├── 重要报告.docx
│   └── 重要报告.pin    ← 置顶标记
```

### 置顶整个合集

在合集文件夹内创建 `.pin` 文件（不带前缀，跟 `.tags` 一致）：
```
content/
├── 工作/
│   └── 重要合集/
│       ├── .pin           ← 整个合集置顶
│       ├── 01-第一章.md
│       └── 02-第二章.md
```

### 合集内文章置顶

在合集文件夹内用 `文件名.pin` 指定合集内部的置顶文章：
```
content/
├── 工作/
│   └── 某合集/
│       ├── 01-第一章.md
│       ├── 01-第一章.pin  ← 合集内置顶
│       └── 02-第二章.md
```

### .pin 文件内容格式

| 内容 | 效果 |
|------|------|
| 空文件 | 全局置顶，权重 0 |
| `1` | 全局置顶，权重 1（数字越小越靠前） |
| `2 工作` | 仅在「工作」标签内置顶，权重 2 |

多篇置顶时按权重数字从小到大排列，全局置顶优先于标签内置顶。

### 取消置顶

删掉对应的 `.pin` 文件，重新构建即可。

## 六、换电脑

### 情况 A：全新电脑
1. 安装 Node.js + Git（见第一节）
2. 终端执行：
```
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
mkdir content
```
3. 把你的 `content/` 文件夹复制过来
4. 正常操作即可

### 情况 B：另一台电脑已经推送过更新
```
cd test1
git pull
npm install
```
拉取最新代码后继续操作。

### 重要：备份 content/ 文件夹
`content/` 不会上传到 GitHub，换电脑时需要**手动拷贝**这个文件夹（U盘、网盘等）。
这是你唯一需要自己备份的东西。

3. `content/` 里的原始文档**不会上传到 GitHub**，别人看不到
