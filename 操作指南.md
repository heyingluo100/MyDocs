# 操作指南（从零开始）

> 本指南面向完全零基础用户，从注册账号到网站上线，每一步都有详细说明。

---

## 一、注册账号（只需做一次）

### 1.1 注册 GitHub 账号

GitHub 是一个代码托管平台，你的网站代码会存在这里。

1. 打开 https://github.com
2. 点击右上角 **Sign up**
3. 输入邮箱、设置密码、选择用户名
4. 完成邮箱验证
5. 登录后你会看到 GitHub 首页

### 1.2 注册 Vercel 账号（推荐部署方式）

Vercel 是一个免费的网站托管平台，推送代码后自动部署。

1. 打开 https://vercel.com
2. 点击 **Sign Up**
3. 选择 **Continue with GitHub**（用 GitHub 账号登录）
4. 授权 Vercel 访问你的 GitHub
5. 完成，Vercel 账号创建好了

## 二、安装软件（只需做一次）

### 2.1 安装 Node.js

Node.js 是运行本项目的环境。

1. 打开 https://nodejs.org
2. 下载 **LTS 版本**（左边那个绿色按钮）
3. 双击安装，一路点「下一步」即可
4. 验证安装：打开终端（Windows 按 `Win+R`，输入 `cmd` 回车），输入：
```
node -v
```
看到版本号（如 `v20.x.x`）就成功了。

> **什么是终端？** 就是一个黑色/白色的命令输入窗口。Windows 叫「命令提示符」或「PowerShell」，Mac 叫「终端」。后续所有命令都在这里输入。

### 2.2 安装 Git

Git 是代码版本管理工具，用来上传代码到 GitHub。

1. 打开 https://git-scm.com
2. 下载安装，一路点「下一步」
3. 验证安装：终端输入：
```
git -v
```
看到版本号就成功了。

### 2.3 配置 Git 身份（首次使用）

Git 需要知道"是谁在操作"，这样每次提交代码都会记录你的名字。这不是注册账号，只是给本地电脑设一个标识。

终端依次输入下面两行命令（**把引号里的中文换成你自己的信息**）：
```
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

**举个例子**，如果你叫张三，邮箱是 zhangsan@qq.com，就输入：
```
git config --global user.name "zhangsan"
git config --global user.email "zhangsan@qq.com"
```

> 这个名字和邮箱不需要和 GitHub 账号一模一样，但建议保持一致，方便辨认。设置一次就行，以后不用再设。

**查看当前电脑绑定的账号**：如果不确定之前设过没有，终端输入：
```
git config --global user.name
git config --global user.email
```
会显示当前设置的名字和邮箱。如果没有任何输出，说明还没设置，按上面的步骤设一下就行。

## 三、创建项目并部署（只需做一次）

### 3.1 在 GitHub 上创建仓库

1. 登录 GitHub，点击右上角 **+** → **New repository**
2. 填写：
   - Repository name：随便取个名字（如 `my-docs`）
   - 选择 **Private**（私有仓库，代码仅自己可见）
   - **不要**勾选任何初始化选项（不要勾 README、.gitignore）
3. 点击 **Create repository**
4. 你会看到一个页面，上面有仓库地址（如 `https://github.com/你的用户名/my-docs.git`），**先别关，后面要用**

> **为什么选 Private？** 项目代码中包含邀请码哈希值、保护机制配置等敏感信息，公开仓库意味着任何人都能看到这些内容。私有仓库完全免费，Vercel 也支持从私有仓库部署，不影响任何功能。

### 3.2 下载项目模板到本地

打开终端，逐行输入：
```
git clone https://github.com/heyingluo100/mydocs.git my-docs && cd my-docs
```

> 这会把项目模板下载到你电脑上的 `my-docs` 文件夹里。

### 3.3 安装依赖

```
npm install
```
等待安装完成（可能需要几分钟，看到没有红色报错就行）。

### 3.4 关联到你自己的 GitHub 仓库

```
git remote set-url origin https://github.com/你的用户名/my-docs.git
```

> 把上面的地址换成你在 3.1 步看到的那个仓库地址。

### 3.5 创建 content 目录并添加第一篇文档

```
mkdir content
```

然后在 `content/` 里创建一个文件夹（就是分类标签），把你的文档放进去。

例如：在 `content/` 下新建一个 `公告` 文件夹，里面放一个 `欢迎.md` 文件，内容随便写点。

### 3.6 首次构建并推送

```
npm run build && git add . && git commit -m "首次提交" && git push -u origin master
```

> 如果提示输入用户名密码，**密码栏要填 Token，不是你的 GitHub 登录密码**。Token 是什么、怎么获取，见下一步。

### 3.7 获取 GitHub Personal Access Token

GitHub 不允许用登录密码推送代码，需要生成一个专用的 Token（令牌）来代替密码。**只需做一次**，生成后保存好反复使用。

**生成步骤：**

1. 登录 GitHub，点击右上角你的**头像** → **Settings**
2. 左侧菜单拉到最底部，点击 **Developer settings**
3. 点击左侧 **Personal access tokens** → **Tokens (classic)**
4. 点击右上角 **Generate new token** → 选 **Generate new token (classic)**
5. 填写：
   - **Note**：随便写个备注（如 `my-docs 推送用`），方便以后辨认
   - **Expiration**：选 **No expiration**（永不过期），或按需选择有效期
   - **勾选权限**：只需勾选 **`repo`**（第一个大类，打勾后下面的子项会自动全选）
6. 拉到页面最底部，点击 **Generate token**
7. **立刻复制 Token！** 页面只显示这一次，关掉就看不到了

> **Token 长什么样？** 以 `ghp_` 开头的一串字母数字，如 `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`。

**使用方式：**
- 终端提示输入 `Password` 时，粘贴这个 Token（粘贴时屏幕不会显示任何字符，这是正常的，直接按回车）
- Windows 用户首次输入后系统会自动记住，以后推送不用再输
- 如果 Token 丢了或过期了，回到上面的页面重新生成一个即可

### 3.8 方式 A：用 Vercel 部署（推荐）

1. 打开 https://vercel.com/dashboard
2. 点击 **Add New...** → **Project**
3. 找到你刚才推送的仓库（如 `my-docs`），点击 **Import**
4. 设置页面保持默认，直接点 **Deploy**
5. 等待 1-2 分钟，部署完成
6. Vercel 会给你一个网址（如 `https://my-docs-xxx.vercel.app`），这就是你的网站地址！

以后每次 `git push`，Vercel 会自动重新部署，不用再手动操作。

### 3.9 方式 B：用 GitHub Pages 部署

> **注意**：GitHub Pages 免费版**不支持私有仓库**。如果你在 3.1 步选了 Private（推荐），请使用方式 A（Vercel）部署。只有 GitHub Pro 付费用户才能用私有仓库部署 GitHub Pages。

1. 打开你的 GitHub 仓库页面
2. 点击 **Settings** → 左侧 **Pages**
3. Source 选择 **GitHub Actions**
4. 项目已经包含了自动部署的配置文件（`.github/workflows/deploy.yml`），推送代码后会自动构建部署
5. 等待几分钟，网站地址为 `https://你的用户名.github.io/my-docs/`

### 3.10 修改网站名称

网站默认叫「文档库」，如果你想改名（比如改成「小明的笔记」），需要改 3 个地方：

| 文件 | 位置 | 改什么 |
|------|------|--------|
| `index.html` | 第 9 行 `<title>` | 浏览器标签页上显示的标题 |
| `src/components/AppHeader.vue` | 搜索「文档库」 | 网站左上角显示的名称 |
| `src/App.vue` | 搜索「Personal Docs」 | 网站底部的版权文字 |

改完后重新构建发布即可。

> 邀请码验证页面（`src/components/InvitationGate.vue`）上也有一行「Personal Docs」，如果开了全站锁也一起改掉。

## 四、日常操作

### 添加 / 修改文档

1. 在 `content/` 下创建文件夹（文件夹名 = 分类标签）
2. 把 `.docx`、`.md` 或 `.txt` 文件拖进去
3. 完成！

**目录结构示例：**
```
content/
├── 工作/                ← 分类「工作」
│   ├── 周报.docx        ← Word 文档
│   └── 项目总结.md      ← Markdown 文档
├── 学习/                ← 分类「学习」
│   └── 笔记.docx
├── 公告/
│   ├── 欢迎.md          ← 单独文章
│   └── 版本记录/        ← 合集（子文件夹 = 合集）
│       ├── v0.1.md
│       └── v0.2.md
└── P3/
    └── P345联动合集/    ← 跨标签合集
        ├── .tags        ← 内容: P4\nP5（额外标签）
        ├── 01-第一章.md
        └── 02-第二章.md
```

### 合集功能

在标签文件夹下创建**子文件夹**，即为合集。合集内文章按文件名排序。

### 多标签（标签穿插）

文章或合集需要出现在多个标签下时，创建 `.tags` 文件指定额外标签：

- **合集多标签**：在合集文件夹内新建 `.tags` 文件（文件名就叫 `.tags`），每行写一个额外标签名
- **单独文章多标签**：在同级目录新建 `文件名.tags` 文件（如 `周报.tags` 对应 `周报.docx`），每行写一个额外标签名

**注意**：合集用的是 `.tags`（不带前缀），单独文章用的是 `文件名.tags`（带文件名前缀），别搞混！

### 本地预览

```
npm run dev
```

浏览器打开显示的地址即可预览。文件有变动会自动更新，无需手动刷新。

### 发布上线

```
npm run build && git add . && git commit -m "更新文档" && git push
```
推送后 Vercel/GitHub Pages 自动部署，1-2 分钟后网站更新。

## 六、邀请码系统

网站支持两层邀请码保护，默认关闭，按需开启。

### 生成邀请码

```
npm run gen-code
```

输出邀请码（明文，发给受邀用户）和哈希值（填入配置）。也可以自定义码：

```
npm run gen-code "你想要的码"
```

### 开启全站锁

全站锁开启后，访问任何页面都必须先输入邀请码。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 打开 `src/config/invitation.js`，修改：
```js
global: {
  enabled: true,
  codeHash: '粘贴哈希值到这里',
}
```
3. 重新构建发布
4. 把邀请码发给受邀用户

### 锁定单篇文章

单篇锁可以给特定文章设独立邀请码，每篇码可以不同。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 在文章旁创建同名 `.lock` 文件，内容为哈希值：
```
content/
├── 工作/
│   ├── 机密报告.docx
│   └── 机密报告.lock    ← 内容为哈希值（一行）
```
3. 打开 `src/config/invitation.js`，设 `article.enabled: true`
4. 重新构建发布
5. 该文章的专属邀请码发给特定用户

### 锁定整个合集

在合集文件夹内放 `.lock` 文件（和 `.tags` 同级），合集内所有文章自动锁住，共享同一个邀请码。

1. 运行 `npm run gen-code`，记下邀请码和哈希值
2. 在合集文件夹内创建 `.lock` 文件，内容为哈希值：
```
content/
├── 工作/
│   └── 机密合集/
│       ├── .lock            ← 内容为哈希值，整个合集一把锁
│       ├── 01-第一章.md
│       └── 02-第二章.md
```
3. 打开 `src/config/invitation.js`，设 `article.enabled: true`
4. 重新构建发布

如果合集内某篇文章还有自己的 `.lock` 文件（如 `01-第一章.lock`），那篇文章优先用自己的码，其余用合集的码。

### 关闭邀请码

把 `src/config/invitation.js` 里对应的 `enabled` 改为 `false`，删掉 `.lock` 文件（可选），重新构建即可。

### 注意

- 邀请码验证后 **7 天有效**，到期需重新输入
- 全站锁和单篇锁可以同时开启（双层保护）
- 配置文件中只存哈希值，不存明文码
- `.lock` 文件里也是哈希值，可以安全提交到 Git

## 七、文档置顶

支持将重要文档或合集固定在列表顶部，支持全局置顶和标签内置顶。

### 置顶单篇文章

在文章旁创建同名 `.pin` 文件：
```
content/
├── 工作/
│   ├── 重要报告.docx
│   └── 重要报告.pin    ← 置顶标记
```

### 置顶整个合集

在合集文件夹内创建 `.pin` 文件（不带前缀，跟 `.tags` 一致）：
```
content/
├── 工作/
│   └── 重要合集/
│       ├── .pin           ← 整个合集置顶
│       ├── 01-第一章.md
│       └── 02-第二章.md
```

### 合集内文章置顶

在合集文件夹内用 `文件名.pin` 指定合集内部的置顶文章：
```
content/
├── 工作/
│   └── 某合集/
│       ├── 01-第一章.md
│       ├── 01-第一章.pin  ← 合集内置顶
│       └── 02-第二章.md
```

### .pin 文件内容格式

| 内容 | 效果 |
|------|------|
| 空文件 | 全局置顶，权重 0 |
| `1` | 全局置顶，权重 1（数字越小越靠前） |
| `2 工作` | 仅在「工作」标签内置顶，权重 2 |

多篇置顶时按权重数字从小到大排列，全局置顶优先于标签内置顶。

### 取消置顶

删掉对应的 `.pin` 文件，重新构建即可。

## 八、换电脑

### 情况 A：全新电脑
1. 安装 Node.js + Git（见第二章）
2. 终端执行：
```
git clone https://github.com/你的用户名/你的仓库名.git && cd 你的仓库名 && npm install && mkdir content
```
3. 把你的 `content/` 文件夹复制过来
4. 正常操作即可

### 情况 B：另一台电脑已经推送过更新
```
cd 你的仓库名 && git pull && npm install
```
拉取最新代码后继续操作。

### 重要：备份 content/ 文件夹
`content/` 不会上传到 GitHub，换电脑时需要**手动拷贝**这个文件夹（U盘、网盘等）。
这是你唯一需要自己备份的东西。

## 九、注意事项

1. **只操作 `content/` 目录**，其他文件不要动
2. 支持的文件格式：`.docx`（Word）、`.doc`（旧版 Word）、`.md`（Markdown）、`.txt`（纯文本）、`.pdf`（附件预览）
3. `content/` 里的原始文档**不会上传到 GitHub**，别人看不到源文件
4. 访客只能通过网站浏览，无法复制、抓取文字内容
5. 每次更新文档后记得执行「发布上线」命令
6. 特殊文件说明：
   - `.tags` → 多标签标记
   - `.lock` → 邀请码锁定
   - `.pin` → 置顶标记

## 十一、修改网站域名

Vercel 默认分配的域名比较随机（如 `my-docs-a1b2c3.vercel.app`），不好记也不好看。你可以免费改成一个好看的 `.vercel.app` 域名。

### 11.1 修改 Vercel 分配的域名（免费）

1. 打开 https://vercel.com/dashboard
2. 点击你的项目
3. 点击顶部 **Settings** → 左侧 **Domains**
4. 你会看到当前的域名（如 `my-docs-a1b2c3.vercel.app`）
5. 点击旁边的 **Edit**，把域名改成你想要的名字（如 `xiaoming-docs.vercel.app`）
6. 点击 **Save**，立即生效

> **注意**：`.vercel.app` 前面的名字全球唯一，如果被别人占了就换一个。建议用 `你的名字-docs` 这种格式。

### 11.2 绑定自己的域名（需购买域名）

如果你有自己的域名（如 `docs.example.com`），也可以绑定到 Vercel：

**Vercel 项目绑定域名：**

1. 打开 Vercel 项目 → **Settings** → **Domains**
2. 输入你的域名（如 `docs.example.com`），点击 **Add**
3. Vercel 会提示你去域名注册商后台添加一条 DNS 记录：
   - 子域名（如 `docs.example.com`）：添加 **CNAME** 记录，值为 `cname.vercel-dns.com`
   - 根域名（如 `example.com`）：添加 **A** 记录，值为 `76.76.21.21`
4. 添加好 DNS 记录后，等待几分钟到几小时生效
5. 生效后 Vercel 会自动配置 HTTPS 证书

**GitHub Pages 绑定域名：**

1. 打开 GitHub 仓库 → **Settings** → **Pages**
2. 在 **Custom domain** 里输入你的域名，点 **Save**
3. 去域名注册商后台添加 DNS 记录：CNAME 记录指向 `你的用户名.github.io`
4. 等待生效，GitHub 会自动配置 HTTPS

> **没有域名？** 完全不需要买！直接用 11.1 的方法改一个好看的 `.vercel.app` 域名就行，免费且够用。
