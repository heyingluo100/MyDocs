# 操作指南

## 一、环境准备（只需做一次）

### 1. 安装 Node.js
- 下载地址：https://nodejs.org
- 选择 LTS 版本，一路下一步安装即可
- 安装完成后打开终端输入 `node -v`，能看到版本号就成功了

### 2. 安装 Git
- 下载地址：https://git-scm.com
- 一路下一步安装即可
- 安装完成后打开终端输入 `git -v`，能看到版本号就成功了

### 3. 获取项目
```bash
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
```

---

## 二、日常操作

### 添加新文档

1. 在 `content/` 目录下创建文件夹（文件夹名 = 分类标签）
2. 把 `.docx` 或 `.md` 文件拖进去
3. 完成！

**目录结构示例：**
```
content/
├── 工作/                ← 分类：工作
│   ├── 周报.docx        ← Word 文档
│   └── 项目总结.md      ← Markdown 文档
├── 学习/                ← 分类：学习
│   └── 笔记.docx
└── 公告/
    └── 欢迎.md
```

### 本地预览

```bash
npm run dev
```
浏览器打开 http://localhost:5173 即可预览。
文件有变动会自动更新，无需手动刷新。

### 部署上线

```bash
npm run build
git add .
git commit -m "更新文档"
git push
```
推送后 GitHub Actions 会自动部署，几分钟后网站就更新了。

网站地址：`https://heyingluo100.github.io/test1/`

## 三、常用操作速查

| 操作 | 方法 |
|------|------|
| 添加文档 | 往 `content/文件夹名/` 里拖入 `.docx` 或 `.md` 文件 |
| 新建分类 | 在 `content/` 下新建一个文件夹 |
| 删除文档 | 直接删除 `content/` 里对应的文件 |
| 删除分类 | 直接删除 `content/` 里对应的文件夹 |
| 本地预览 | `npm run dev` |
| 部署上线 | `git add . && git commit -m "更新" && git push` |
| 换电脑使用 | 新电脑装 Node.js + Git → `git clone` → `npm install` |

---

## 四、换电脑操作

### 在新电脑上获取项目
```bash
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
```

### 如果另一台电脑已经推送过更新
```bash
cd test1
git pull
```
拉取最新代码后即可继续操作。

---

## 五、注意事项

1. **不要删除** `src/`、`scripts/`、`vite.config.js` 等文件，它们是项目核心
2. **只操作 `content/` 目录**，其他文件不要动
3. 仓库是 **Private（私有）** 的，只有你能看到源码
4. 别人只能通过网站链接浏览内容，无法看到你的原始文件
5. 支持的文件格式：`.docx`（Word）、`.md`（Markdown）、`.pdf`（附件预览）
