# 操作指南

## 一、新电脑首次部署（只需做一次）

### 第 1 步：安装 Node.js
1. 打开 https://nodejs.org
2. 下载 **LTS 版本**（左边那个按钮）
3. 一路下一步安装
4. 打开终端（Windows 按 `Win+R` 输入 `cmd` 回车），输入：
```
node -v
```
看到版本号（如 `v20.x.x`）就成功了。

### 第 2 步：安装 Git
1. 打开 https://git-scm.com
2. 下载安装，一路下一步
3. 终端输入：
```
git -v
```
看到版本号就成功了。

### 第 3 步：下载项目
打开终端，逐行输入：
```
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
```
等依赖安装完成（可能需要几分钟）。

### 第 4 步：创建 content 目录
项目下载后没有 `content/` 目录（它不上传到 GitHub），需要自己建：
```
mkdir content
```
然后在 `content/` 里创建分类文件夹，把文档拖进去即可。

如果你有之前电脑上的 `content/` 文件夹备份，直接复制过来放到项目根目录下。

## 二、日常操作

### 添加 / 修改文档

1. 在 `content/` 下创建文件夹（文件夹名 = 分类标签）
2. 把 `.docx`、`.md` 或 `.txt` 文件拖进去
3. 完成！

**目录结构示例：**
```
content/
├── 工作/                ← 分类「工作」
│   ├── 周报.docx        ← Word 文档
│   └── 项目总结.md      ← Markdown 文档
├── 学习/                ← 分类「学习」
│   └── 笔记.docx
├── 公告/
│   ├── 欢迎.md          ← 单独文章
│   └── 版本记录/        ← 合集（子文件夹 = 合集）
│       ├── v0.1.md
│       └── v0.2.md
└── P3/
    └── P345联动合集/    ← 跨标签合集
        ├── .tags        ← 内容: P4\nP5（额外标签）
        ├── 01-第一章.md
        └── 02-第二章.md
```

### 合集功能

在标签文件夹下创建**子文件夹**，即为合集。合集内文章按文件名排序。

### 多标签（标签穿插）

文章或合集需要出现在多个标签下时，创建 `.tags` 文件指定额外标签：

- **合集多标签**：在合集文件夹内新建 `.tags` 文件，每行写一个额外标签名
- **单独文章多标签**：在同级目录新建 `文件名.tags` 文件（如 `周报.tags`），每行写一个额外标签名

文件只存一份，不需要复制。

### 本地预览

```
npm run dev
```
浏览器打开 http://localhost:5173 即可预览。
文件有变动会自动更新，无需手动刷新。

### 发布上线（3 步）

```
npm run build
git add .
git commit -m "更新文档"
git push
```
推送后 Vercel 自动部署，1-2 分钟后网站更新。

网站地址：https://test1-dun-pi.vercel.app

## 三、常用操作速查

| 我想要... | 怎么做 |
|-----------|--------|
| 添加文档 | 往 `content/分类名/` 里拖入 `.docx`、`.md` 或 `.txt` 文件 |
| 新建分类 | 在 `content/` 下新建一个文件夹 |
| 新建合集 | 在分类文件夹下新建子文件夹，放入文档 |
| 多标签 | 在合集文件夹内或文件旁新建 `.tags` 文件，每行写一个额外标签 |
| 删除文档 | 删除 `content/` 里对应的文件 |
| 删除分类 | 删除 `content/` 里对应的文件夹 |
| 本地预览 | 终端运行 `npm run dev` |
| 发布上线 | 终端运行下面 4 行命令（见下方） |
| 拉取最新 | 终端运行 `git pull` |

**发布上线完整命令（复制粘贴即可）：**
```
npm run build && git add . && git commit -m "更新文档" && git push
```

## 四、换电脑

### 情况 A：全新电脑
1. 安装 Node.js + Git（见第一节）
2. 终端执行：
```
git clone https://github.com/heyingluo100/test1.git
cd test1
npm install
mkdir content
```
3. 把你的 `content/` 文件夹复制过来
4. 正常操作即可

### 情况 B：另一台电脑已经推送过更新
```
cd test1
git pull
npm install
```
拉取最新代码后继续操作。

### 重要：备份 content/ 文件夹
`content/` 不会上传到 GitHub，换电脑时需要**手动拷贝**这个文件夹（U盘、网盘等）。
这是你唯一需要自己备份的东西。

## 五、注意事项

1. **只操作 `content/` 目录**，其他文件不要动
2. 支持的文件格式：`.docx`（Word）、`.md`（Markdown）、`.pdf`（附件预览）
3. `content/` 里的原始文档**不会上传到 GitHub**，别人看不到
