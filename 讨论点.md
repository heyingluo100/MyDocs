# 文档/合集更新标签方案讨论

## 需求概述
当文档或合集有更新时，在卡片上显示小标签提醒用户。

---

## 需要确定的问题

### 1. "更新"的判定条件
- `updatedAt !== createdAt` 即视为有更新

### 2. 标签消失机制（核心问题）

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A. 时间窗口自动消失** | 零成本，无需记录状态 | 用户可能已经看过了还在显示 |
| 例：更新后 7 天内显示 | 实现简单 | 时间阈值不好定 |
| **B. 用户已读消失** | 最精准，看过就消 | 需要 localStorage 记录每篇的已读时间 |
| 点进文章后标记为已读 | 体验最好 | 换浏览器/清缓存后重置 |
| **C. A+B 混合** | 兼顾两者 | 实现稍复杂 |
| 点进去消失，最多保留 N 天 | 没看过的也不会永远挂着 | |

### 3. 新建文档是否也打标签？

| 方案 | 说明 |
|------|------|
| **只标"更新"** | 更新显示「已更新」，新建的不打标 |
| **区分"新"和"更新"** | 新建 N 天内显示「新」，更新显示「已更新」 |
| **统一标"新动态"** | 不区分新建和更新，统一一个标签 |

### 4. 合集的更新判定

| 方案 | 说明 |
|------|------|
| **任一文章更新 = 合集更新** | 合集内有一篇更新了就打标 |
| **合集新增文章才算更新** | 只有新增文章时才打标，修改内容不算 |

### 5. 标签样式方向

| 样式 | 描述 |
|------|------|
| **小圆点** | 卡片右上角一个小彩色圆点，最克制 |
| **文字标签** | 「已更新」/「新」小药丸标签，信息更明确 |
| **圆点 + 悬停文字** | 默认显示小圆点，hover 时展开为文字 |

## 最终确认

| 维度 | 结论 |
|------|------|
| 更新判定 | `updatedAt !== createdAt` |
| 消失机制 | 已读消失（localStorage 记录点进文章的时间） |
| 新建标签 | 创建 3 天内显示绿色圆点，超期自动消失 |
| 合集更新 | 仅新增文章时才算更新 |
| 样式 | 卡片右上角小圆点，新=绿色，更新=橙色 |
