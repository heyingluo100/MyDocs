# 个人文档库 — 产品需求文档（PRD）

## 1. 产品概述

| 项目 | 说明 |
|------|------|
| 产品名称 | 个人文档库 |
| 产品类型 | 静态个人文档存档网站 |
| 目标用户 | 站主（管理）+ 访客（只读浏览） |
| 部署方式 | GitHub Pages / Vercel（纯静态） |
| 技术栈 | Vue 3 + Vite + Tailwind CSS 4 |

## 2. 核心需求

### 2.1 内容管理（文件夹即标签）
- 在 `content/` 下创建文件夹，**文件夹名就是分类标签**
- 将 `.docx`（Word）或 `.md`（Markdown）文件拖入文件夹即完成发布
- `.pdf` 等附件文件会自动复制到公开目录供预览
- Word 文件通过 mammoth.js 在构建时转为 HTML
- 构建时自动扫描所有分类文件夹，生成文章索引 JSON
- 示例：`content/工作/报告.docx` → 标签为「工作」，标题为「报告」

### 2.2 内容展示
- 首页展示所有文章卡片列表（标题、摘要、标签、日期）
- 侧栏标签系统，类似文件夹分类，点击筛选
- 文章详情页渲染 Markdown 内容（标题、段落、代码、表格、引用等）
- 附件文件预览（PDF 使用 iframe 预览）

### 2.3 内容保护（高强度）
- **CSS 层**：禁止文字选择、禁止拖拽、打印时隐藏内容
- **JS 层**：禁用右键菜单、拦截复制/查看源码/打印/DevTools 快捷键
- **水印层**：Canvas 动态水印覆盖全屏，显示时间戳 + 随机 ID
- **反调试层**：DevTools 尺寸检测 + debugger 时间差检测，检测到则模糊内容
- **内容加载保护**：内容 Base64 编码存储，JS 动态解码渲染
- **标签页检测**：切换标签页时内容模糊，防止后台截图
- **爬虫防护**：robots.txt 全站禁止、meta 标签禁止索引和缓存

### 2.4 主题系统
- 支持亮色/暗色主题切换
- 优先跟随系统偏好，支持手动切换
- localStorage 持久化用户选择

### 2.5 访问模型
- 网站完全公开，无需登录/密码
- 所有访客只能查看，不能复制/修改/下载文字内容
- 站主通过本地文件管理 + 部署更新

## 3. 非功能需求

| 需求 | 说明 |
|------|------|
| 性能 | 首屏加载 < 2s，文章切换流畅 |
| 兼容性 | Chrome/Edge/Firefox/Safari 最新版 |
| 响应式 | 支持桌面和移动端 |
| SEO | 故意禁止 SEO（robots.txt + meta noindex），保护内容 |

## 4. 用户故事

### 站主
1. 我在 `content/` 下创建分类文件夹（如 `content/工作/`）
2. 我把 Word 文件拖进文件夹
3. 我运行 `npm run build` 构建静态站点
4. 我部署到 GitHub Pages / Vercel
5. 访客打开网站即可浏览我的文档

### 访客
1. 我打开网站，看到文章列表
2. 我点击左侧标签，按分类筛选文章
3. 我点击文章卡片，查看文章详情
4. 我尝试复制文字 → 无法选中
5. 我尝试右键 → 菜单被拦截
6. 我尝试 Ctrl+C → 快捷键被拦截
7. 我尝试打开 DevTools → 内容变模糊
8. 我切换标签页 → 内容变模糊
