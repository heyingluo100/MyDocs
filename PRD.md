# 个人文档库 — 产品需求文档（PRD）

## 1. 产品概述

| 项目 | 说明 |
|------|------|
| 产品名称 | 个人文档库 |
| 产品类型 | 静态个人文档存档网站 |
| 目标用户 | 站主（管理）+ 访客（只读浏览） |
| 部署方式 | GitHub Pages / Vercel（纯静态） |
| 技术栈 | Vue 3 + Vite + Tailwind CSS 4 |

## 2. 核心需求

### 2.1 内容管理（文件夹即标签）
- 在 `content/` 下创建文件夹，**文件夹名就是分类标签**
- 将 `.docx`（Word）或 `.md`（Markdown）文件拖入文件夹即完成发布
- `.doc`（旧版 Word 97-2003）文件通过 word-extractor 提取文本并渲染
- `.txt` 文本文件内容直接渲染在页面上
- `.pdf` 等附件文件会自动复制到公开目录供预览
- Word 文件通过 mammoth.js 在构建时转为 HTML，**精确保留空行和段落间距**
- 文档中的空格和换行在页面上完整保留（`white-space: pre-wrap`）
- 构建时自动裁掉文档末尾空段落，保留文中空行
- 构建时自动扫描所有分类文件夹，生成文章索引 JSON
- 自动跳过 Word 临时文件（`~$` 开头）
- **合集功能**：标签文件夹下的子目录自动识别为合集，合集内文章按文件名排序
- **多标签**：合集文件夹内放 `.tags` 文件（文件名就叫 `.tags`），单独文件旁放 `文件名.tags`（如 `报告.tags` 对应 `报告.docx`），每行写一个额外标签名
- **双时间字段**：创建时间（fs.birthtime / .docx 内部元数据）+ 更新时间（mtime），按创建时间倒序排列
- 示例：`content/工作/报告.docx` → 标签为「工作」，标题为「报告」

### 2.2 内容展示
- 首页展示所有文章卡片列表（标题、摘要、标签、日期、合集标记）
- 合集以堆叠卡片样式混排在文章网格中
- 文章按创建时间倒序排列
- 弹出面板式标签筛选（桌面弹出面板 + 移动端底部抽屉）
- 全局标题搜索，跟随当前标签/合集上下文
- 文章详情页渲染 Markdown 内容（标题、段落、代码、表格、引用等）
- 文章详情页顶部阅读进度条 + 滚动触发 sticky 导航栏
- 阅读位置记忆，7天内再次打开弹窗提示继续阅读
- 文章底部上一篇/目录/下一篇导航（合集内隔离导航，单独文章排除合集）
- 合集详情页展示合集内所有文章卡片
- 页面切换 fade 过渡动画 + 合集页卡片展开动画
- 附件文件预览（PDF 使用 iframe 预览）

### 2.3 实时更新系统
- 开发模式下，后台修改文档时自动检测变化并推送到浏览器
- **正在查看的文章被修改时**，弹窗提醒"文档已更新"
- **阅读历史记录**：用 localStorage 记录每篇文章的阅读版本，再次打开已更新文章时弹窗提醒
- SPA 内导航到已更新文章时弹窗提醒，刷新页面时直接显示最新内容（不弹窗）
- 文章被删除时弹窗提醒并引导返回首页

### 2.4 内容保护（高强度）
- **CSS 层**：禁止文字选择、禁止拖拽、打印时隐藏内容
- **JS 层**：禁用右键菜单、拦截复制/查看源码/打印/DevTools 快捷键
- **水印层**：Canvas 动态水印覆盖全屏，显示时间戳 + 随机 ID
- **反调试层**：DevTools 尺寸检测 + debugger 时间差检测，检测到则模糊内容
- **内容加载保护**：内容 Base64 编码存储，JS 动态解码渲染
- **标签页检测**：切换标签页时内容模糊，防止后台截图
- **爬虫防护**：robots.txt 全站禁止、meta 标签禁止索引和缓存

### 2.5 主题系统
- 支持亮色/暗色主题切换
- 优先跟随系统偏好，支持手动切换
- localStorage 持久化用户选择

### 2.6 访问模型
- 网站完全公开，无需登录/密码
- 所有访客只能查看，不能复制/修改/下载文字内容
- 站主通过本地文件管理 + 部署更新

## 3. 非功能需求

| 需求 | 说明 |
|------|------|
| 性能 | 首屏加载 < 2s，文章切换流畅 |
| 兼容性 | Chrome/Edge/Firefox/Safari 最新版 |
| 响应式 | 支持桌面和移动端 |
| SEO | 故意禁止 SEO（robots.txt + meta noindex），保护内容 |

## 4. 用户故事

### 站主
1. 我在 `content/` 下创建分类文件夹（如 `content/工作/`）
2. 我把 Word 文件拖进文件夹
3. 我运行 `npm run build` 构建静态站点
4. 我部署到 GitHub Pages / Vercel
5. 访客打开网站即可浏览我的文档

### 访客
1. 我打开网站，看到文章列表
2. 我点击左侧标签，按分类筛选文章
3. 我点击文章卡片，查看文章详情
4. 我尝试复制文字 → 无法选中
5. 我尝试右键 → 菜单被拦截
6. 我尝试 Ctrl+C → 快捷键被拦截
7. 我尝试打开 DevTools → 内容变模糊
8. 我切换标签页 → 内容变模糊
